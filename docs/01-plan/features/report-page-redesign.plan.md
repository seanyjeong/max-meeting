# Plan: 인쇄용 회의록 페이지 개선

## 1. 개요

| 항목 | 내용 |
|------|------|
| Feature | report-page-redesign |
| 작성일 | 2026-01-31 |
| 우선순위 | High |
| 예상 범위 | Frontend Only |

## 2. 현재 문제점

### 2.1 레이아웃 문제
- **너비 부족**: `max-width: 800px`로 화면에서 너무 좁아 보임
- **A4 활용 미흡**: 인쇄 시 여백이 많이 남음
- **정보 밀도 낮음**: 공간 활용이 비효율적

### 2.2 기능 누락
- **포스트잇 메모 표시 없음**: 결과 페이지에는 있지만 회의록에는 없음
- **스케치 인쇄 불가**: `no-print` 클래스로 인쇄에서 제외됨

## 3. 개선 목표

### 3.1 레이아웃 개선
| 항목 | 현재 | 목표 |
|------|------|------|
| 화면 너비 | 800px | 1000px (화면), A4 full (인쇄) |
| 여백 | 과다 | 적절히 조정 |
| 정보 밀도 | 낮음 | 2단 구성으로 개선 |

### 3.2 기능 추가
- [x] 포스트잇 스타일 메모 표시
- [x] 스케치 썸네일 표시 (인쇄 포함)
- [x] 2단 레이아웃: 왼쪽(안건/토론) + 오른쪽(메모/필기)

## 4. 구현 계획

### Phase 1: 레이아웃 확장
```
변경 파일: report/+page.svelte

1. max-width 조정
   - 화면: 1000px → 1100px
   - 인쇄: A4 full width

2. 2단 레이아웃 구조
   ┌─────────────────────────────────────┐
   │            회의록 헤더               │
   ├─────────────────────────────────────┤
   │            회의 요약                 │
   ├───────────────────┬─────────────────┤
   │                   │                 │
   │    안건/토론      │   메모/필기     │
   │    (60%)          │   (40%)         │
   │                   │                 │
   ├───────────────────┴─────────────────┤
   │            실행 항목                 │
   └─────────────────────────────────────┘
```

### Phase 2: 포스트잇 메모 추가
```
컴포넌트 재사용: PostItNote.svelte

- 안건별 메모를 포스트잇 스타일로 표시
- 인쇄 시에도 색상 유지
```

### Phase 3: 스케치 인쇄 지원
```
변경 사항:
- no-print 제거
- 썸네일 이미지로 표시
- 인쇄 시 적절한 크기로 조정
```
